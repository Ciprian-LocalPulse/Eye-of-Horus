# Style Guide

Status: initial.

## Documentation

- Use precise language.
- Mark TODOs explicitly.
- Separate implemented features from planned features.
- Avoid hype and unverifiable claims.

## Rust

- Run `cargo fmt --all`.
- Prefer small modules with tests.
- Do not introduce dependencies without a clear purpose.

## Eye of Horus Examples

- Use `.eoh` files.
- Add comments when behavior is illustrative.
- Do not imply placeholder examples are executable until the parser exists.
