# RFC Process

Significant design changes require an RFC.

## Requires an RFC

- syntax changes;
- semantic rules;
- memory model changes;
- runtime scheduling;
- standard-library design;
- compatibility policy.

## Process

1. Copy `rfcs/0000-template.md`.
2. Choose a descriptive title.
3. Explain motivation, design, drawbacks, and alternatives.
4. Open a pull request.
5. Discuss until accepted, revised, postponed, or rejected.
